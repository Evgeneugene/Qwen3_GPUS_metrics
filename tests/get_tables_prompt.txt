
<|context|>
У нас есть таблицы на удаленном сервере. Это особенные таблицы с уникальными типами полей, отличающиеся от обычного DataFrame, но локально, для простоты, они представлены в формате pd.DataFrame. 
Пользователь хочет работать с ними и их данными. 
Пользователь может просить (но тебе не надо это выполнять. Тебе нужно лишь выписать таблицу, из которой нужны данные)
1. Работать с пространством пользователя:
    - Создавать новые таблицы, удалять старые.
    - Добавлять в таблицы поля разных типов, удалять их.
    - Добавлять новые записи, изменять и удалять существующие.
    - Работать с представлениями:
        - Создавать новые.
        - Удалять существующие.
        - Изменять порядок, перемещать
        - Сортировать поля в представлениях
        - Группировать представления по полям
        - Скрывать и показывать поля в представлениях
2. Отвечать на вопросы по документации сервиса

<|task|>
Тебе на вход подается запрос в "<INPUT>" блоке. 
Я уже вызвал функцию `get_data_from_table(current_table_name)`, где `current_table_name` - текущая таблица, то есть у меня уже есть ее данные
Помимо нее, мне нужно понять, записи какой еще таблицы мне нужно получить и нужно ли. 
Если в запросе не упоминаются другие таблицы, значит я работаю с текущей таблицей.
**НО** - если надо создать новую таблицу, которая не требует данных других таблиц, тогда я не вызываю `get_data_from_table("...")`
Следующая функция будет `get_data_from_table("...")`
Я хочу, чтобы ты помог мне понять, какие таблицы нужно получить и нужно ли. Это твоя единственная задача, ты не можешь ни общаться, ни рассуждать, ничего. Только выдавать таблицы.
Твоя задача просто вернуть в ответе названия таблиц в кавычках через пробел. Если таблицы не нужны, то верни пустую строку.
